## 핵심 원칙

피드백 루프는 강력하지만, **자동화할 부분과 사람이 판단할 부분을 분리**하지 않으면 노이즈를 증폭시킨다. 스킬에 규칙은 적을수록 좋다.

## Hook 타이밍

| Hook | 시점 | 적합성 |
|------|------|--------|
| `user-prompt-submit` | Claude 응답 전 | 부적합 — 이전 맥락 접근 불가 |
| `stop` | 응답 완료 후 | 적합 — 전체 대화 맥락 접근 가능 |

수정 패턴 감지는 `stop` hook에서 처리한다.

## 자동 업데이트의 함정

- **비대화** — 모든 수정을 스킬에 넣으면 규칙이 빠르게 누적됨
- **잘못된 원인 귀속** — 스킬 부족인지, 모델 추론 실수인지, 기대치 변화인지 구분 필요
- **오탐** — "이걸 바꿔줘" 같은 새 작업 요청을 수정 요청으로 잘못 분류

## 안전한 구조: 자동 감지 + 사람 확인

```
1. stop hook에서 수정 패턴 감지
2. "이 내용을 스킬에 추가할까요?" 확인 요청
3. 승인되면 스킬 파일 수정 + git commit
```

세션 종료 시 일괄 검토하는 방식도 유효하다. 스킬은 항상 git으로 추적해야 롤백 가능.
